#### What REST API endpoints does this server expose, and how do they map to the MySQL tables?  

Based on the provided `server1.py` file, the server exposes a REST API for managing company and stock market data. The API interacts with two main MySQL tables: `companies` and `stocks`.

Here's a summary of the API endpoints and their database interactions:

### MySQL Tables

From the queries in `server1.py`, we can infer the following table structures:

*   **`companies`**: Stores company information.
    *   `ticker` (Primary Key, VARCHAR)
    *   `name` (VARCHAR)
    *   `sector` (VARCHAR)
*   **`stocks`**: Stores daily stock records.
    *   `ticker` (Foreign Key to `companies.ticker`)
    *   `date` (DATE)
    *   `high` (DECIMAL/FLOAT)
    *   `low` (DECIMAL/FLOAT)
    *   (Composite Primary Key on `ticker` and `date`)

---

### API Endpoints

All endpoints are prefixed with `/<db>/api/`, where `<db>` is the name of the database to use (e.g., `prod`).

1.  **Create/Update Company**
    *   **Endpoint:** `POST /companies/<ticker>`
    *   **Description:** Creates a new company or updates an existing one identified by its `ticker`.
    *   **Body:** `{ "name": "...", "sector": "..." }`
    *   **Database Mapping:** Executes an `INSERT ... ON DUPLICATE KEY UPDATE` on the `companies` table.

2.  **Add Stock Record**
    *   **Endpoint:** `POST /companies/<ticker>/records/<date>`
    *   **Description:** Adds or updates a daily stock record for a company.
    *   **Body:** `{ "high": ..., "low": ... }`
    *   **Database Mapping:** Executes an `INSERT ... ON DUPLICATE KEY UPDATE` on the `stocks` table.

3.  **Get All Stock Records for a Company**
    *   **Endpoint:** `GET /companies/<ticker>/records`
    *   **Description:** Retrieves all stock records for a specific `ticker`, joined with company details.
    *   **Database Mapping:** Executes a `SELECT` query that `JOIN`s the `stocks` and `companies` tables on the `ticker` column.

4.  **Get a Specific Stock Record**
    *   **Endpoint:** `GET /companies/<ticker>/records/<date>`
    *   **Description:** Retrieves a single stock record for a given `ticker` and `date`.
    *   **Database Mapping:** Executes a `SELECT` query that `JOIN`s `stocks` and `companies` tables, filtering by `ticker` and `date`.

5.  **Get Stock Records in a Date Range**
    *   **Endpoint:** `GET /companies/<ticker>/records/range?start=...&end=...`
    *   **Description:** Retrieves stock records for a `ticker` within a specified date range.
    *   **Database Mapping:** Executes a `SELECT` query that `JOIN`s `stocks` and `companies` and filters records where the `date` is `BETWEEN` the `start` and `end` query parameters.

6.  **Get Monthly Average Stock Data**
    *   **Endpoint:** `GET /companies/<ticker>/records/monthly`
    *   **Description:** Calculates and returns the average monthly high price for a given `ticker`.
    *   **Database Mapping:** Executes a `SELECT` query on the `stocks` table that uses `GROUP BY` on the month and `AVG()` to calculate the average `high` price.
